# BUG1024 品牌标志设计规范

## 1. 核心定义

`BUG1024` 完整字标是第一品牌资产。它不依赖额外图腾，也不试图把十多年的经历全部压缩成复杂隐喻。

标志只有一个异常：字母 `G` 的开口处有一枚脱离主体的钢蓝色模块。主体代表秩序、系统与长期积累；错位模块代表 Bug，也代表人在系统中保留的独立判断。

品牌识别遵循三层关系：

1. 第一眼读出 `BUG1024`；
2. 第二眼发现 `G` 上的异常；
3. 长期使用后，让这枚异常模块成为 Bug1024 的专属记忆。

## 2. 标志组合

### 完整字标

优先用于网站页眉、文章署名、名片和横向展示。

- 浅色背景：[bug1024-wordmark.svg](/brand/bug1024-wordmark.svg)
- 深色背景：[bug1024-wordmark-light.svg](/brand/bug1024-wordmark-light.svg)

### 独立符号

从字标中的异常 `G` 提取，用于 favicon、头像和空间有限的场景。它是完整字标的缩写，不应在首次接触品牌的大型场景中单独使用。

- 浅色背景：[bug1024-mark.svg](/brand/bug1024-mark.svg)
- 深色背景：[bug1024-mark-light.svg](/brand/bug1024-mark-light.svg)

## 3. 构造规则

- 字形基于统一的几何骨架，笔画粗细一致。
- 转角采用小半径圆角，避免纯机械感，也不形成明显书法风格。
- `G` 是唯一允许出现结构异常的字符。
- 蓝色模块与 `G` 主体保持一个笔画宽度以内的距离，看起来像脱离，又仍属于原结构。
- 任何版本都不得再给其他字符增加缺口、错位或强调色。

## 4. 标准颜色

| 名称 | 色值 | 用途 |
| --- | --- | --- |
| 墨色 Ink | `#111A21` | 浅色背景上的主体 |
| 钢蓝 Steel | `#365B6D` | 唯一异常模块 |
| 雾白 Mist | `#F8FAF9` | 品牌浅色背景 |
| 夜墨 Night | `#11181C` | 深色背景 |
| 反白 Light | `#EDF2F1` | 深色背景上的主体 |
| 浅钢蓝 | `#8EAFBA` | 深色背景上的异常模块 |

钢蓝不是装饰色。没有异常模块的场景，不应为了活跃画面而随意增加钢蓝色。

## 5. 安全留白

以完整字标的标准笔画宽度 `x` 为基准：

- 完整字标四周至少保留 `2x`；
- 独立符号四周至少保留 `1.5x`；
- 不允许其他文字、边框或图形进入安全区域。

## 6. 最小尺寸

- 完整字标：屏幕宽度不小于 `96px`，印刷宽度不小于 `28mm`；
- 独立符号：屏幕不小于 `16px`，印刷不小于 `5mm`；
- 小于 `24px` 时只使用独立符号，不使用完整字标。

## 7. 背景使用

- 雾白和浅色纯色背景使用墨色版本；
- 夜墨和深色纯色背景使用反白版本；
- 复杂图片上不得直接放置标志，应先增加纯色承载区域；
- 不使用投影、描边、渐变或半透明效果增强对比。

## 8. 禁止事项

- 不拉伸、压缩、旋转或倾斜；
- 不改变字符间距和蓝色模块位置；
- 不将蓝色模块换成高饱和霓虹色；
- 不添加昆虫、代码括号、电路、芯片等技术行业符号；
- 不把独立 `G` 误用为完整名称的长期替代；
- 不在同一画面制造第二个“Bug”。

## 9. 品牌语气

标志所对应的表达应保持：具体、克制、有判断，不过度包装。它不是科技公司的效率符号，也不是故作深沉的文化图腾，而是一个使用十多年、仍在继续生长的个人署名。

## 10. 博客应用位置

| 位置 | 使用版本 | 规则 |
| --- | --- | --- |
| 网站页眉 | 完整字标 | 全站固定出现一次，是主要身份入口 |
| 浏览器 favicon | 独立符号 | SVG 优先，ICO/PNG 作为兼容回退 |
| 手机桌面图标 | 独立符号 | 雾白方形底，保留足够安全区 |
| PWA 图标 | 独立符号 | 提供 192px 与 512px maskable 图标 |
| 分享与结构化数据 | 512px 独立符号 | 文章无题图时作为品牌兜底图 |

首页主视觉、文章标题和正文不重复放置标志。品牌识别由页眉持续完成，内容区域应把注意力留给文字。
