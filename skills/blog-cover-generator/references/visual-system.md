# 第 1024 格：BUG1024 视觉系统

## 目录

- 定位与作者签名
- 秩序、Bug 类型与钢蓝作用
- 默认媒介与作者摄影
- 视觉简报与验收
- 外部签名

## 定位

“第 1024 格”是 BUG1024 的生成视觉作品体系。文章封面是它的主要使用场景，但作品应能脱离文章页独立成立。

这套系统把 BUG1024 标志中错位的钢蓝模块转化为可持续使用的视觉语法：

> 一种真实秩序，一个明确的 Bug，一次克制的钢蓝回应。

不要把它变成画面里的口号、重复出现的 Logo，或寻找隐藏数字的游戏。

## 作者签名

每张 AI 生成封面必须包含三部分：

1. **可辨认的秩序**：来自文章的一种真实排列、节奏、边界、顺序、路径、支撑或重复结构；
2. **恰好一个 Bug**：从文章真实矛盾中产生的一次有意义偏差，而不是为了风格临时添加；
3. **明确的钢蓝作用**：`#365B6D` 标记、成为、重新连接、支撑或回应这个 Bug。它必须参与核心关系，不能装饰无关物件。

预期观看顺序：

1. 第一眼看到安静、完整的画面；
2. 第二眼发现其中一处没有完全服从秩序；
3. 经过反复观看后，把这种克制的偏差与 BUG1024 联系起来。

Bug 在 `328 × 246px` 缩略图中仍应能够被发现，但不得抢占第一视觉中心。

## 秩序的来源

优先使用场景中原本存在的秩序，不要叠加通用科技图层：

- 对齐的纸张、书籍、文件夹、抽屉、工具或座位；
- 窗格、地砖、书架、建筑、道路、地图或光影；
- 重复的折痕、接点、缝线、支撑或模块；
- 按时间排列的物件、家庭惯例、工作顺序或空间边界。

只有文章确实涉及规划、建筑、测量、地图或形式系统时，才使用可见网格、坐标、刻度或制图线。不得在每个场景上覆盖通用半透明网格。

## Bug 类型

编写 Prompt 前只选择一种：

| 类型 | 视觉表现 | 常见主题 |
| --- | --- | --- |
| 位移 `offset` | 一个元素偏离预期位置 | 离开、迁移、改变方向 |
| 越界 `overflow` | 一个元素跨出边界或超过容量 | 成长、野心、组织边界 |
| 缺失 `missing` | 预期存在的一个元素消失 | 记忆、失去、时间、残缺记录 |
| 重复 `duplicate` | 原本唯一的元素出现两次 | AI、模仿、身份、复制 |
| 错序 `out-of-order` | 一个元素打破原本可读的顺序 | 人生阶段、历史资料、职业变化 |
| 断裂 `break` | 连续结构中出现一处断点 | 失败、冲突、中断、系统脆弱性 |
| 修补 `patch` | 一处局部修补重新连接或承载更大结构 | 责任、维护、家庭、恢复 |
| 分叉 `fork` | 一条路径出现一个计划外但有意选择的分支 | 判断、选择、不同未来 |

不要为了显得巧妙而合并多个类型。同一画面同时出现异常涟漪、错位纸张、断裂网格和蓝色修补板，是四个异常，不是一个更丰富的概念。

## 钢蓝的作用

选择一种作用，并在视觉简报中明确记录：

- **Bug 本身**：发生位移或例外的元素就是钢蓝色；
- **修补 `patch`**：钢蓝重新连接或支撑被破坏的秩序；
- **痕迹 `trace`**：钢蓝记录偏差发生的位置，但不形成第二个异常。

AI 生成封面中的钢蓝通常应少于画面面积的 10%，最多不得超过 15%。如果删掉钢蓝仍不影响核心关系，它很可能只是装饰，需要重新设计概念。

## 默认生成媒介

默认使用**结构静物（constructed editorial still-life）**：介于真实摄影和精心搭建的编辑模型之间，使用可信的纸张、木材、玻璃、石材、石墨、布料或旧金属。

保留触摸、修补、使用和时间留下的痕迹。避免展厅式完美和通用怀旧感。文章需要时可以选择其他方向，但必须保留“秩序—Bug—钢蓝”的关系。

## 作者摄影

不得为了制造 Bug 而对作者纪实照片虚构、变色、移动、删除或添加物件。真实照片可以通过观察、编排、裁切和展示方式进入视觉作品体系；事实真实性高于签名一致性。

原始照片本身存在与文章有关的偏差时，只需准确描述，不要夸大。必要时让作者摄影与 AI 生成的“第 1024 格”作品保持不同系列。

## 避免字面化

- 不得在生成图中写入 `1024`、`BUG1024`、封面编号或档案编号；
- 不得要求每张图都出现真实格子或网格；“第 1024 格”是系列名称，不是寻找彩蛋；
- 不得为了让概念更容易解释而增加第二个异常；
- 钢蓝物件与 Bug 没有因果关系时，不得把它作为随机点缀；
- 不得用人工科技图层把家庭、记忆或地点文章过度理性化。

## 视觉简报约定

生成前记录：

```text
自然秩序：<什么构成了系统或节奏>
Bug 类型：<从分类中选择一种英文标识>
Bug 表现：<画面中唯一可见的偏差>
钢蓝作用：<Bug、patch 或 trace>
人的痕迹：<可选；使用、选择、照料或维护的证据>
```

任何字段含糊不清，或者可以直接替换到无关文章中时，先继续收紧概念，不要开始生成。

## 作者签名验收

以下任一问题回答“否”时，拒绝或修改方案：

1. 不解释偏差时，画面中的秩序是否已经可见？
2. Bug 是否来自文章的具体主题或矛盾？
3. 是否只有一种 Bug 类型？
4. 缩略图尺寸下是否仍能发现这个 Bug？
5. 钢蓝是否在标记或回应 Bug，而不只是装饰画面？
6. 删除钢蓝元素后，核心关系是否会明显减弱？
7. 读者发现 Bug 之前，画面本身是否仍然完整？
8. 场景是否足够具体，不会同样适用于一篇无关文章？

## 外部签名

文章封面保持无 Logo、无文字。用户需要社交传播版或作品集版本时，通过确定性排版在生成图之外增加署名：

- 官方 BUG1024 字标或独立符号；
- `BUG VISUAL 001` 一类连续编号；
- 年份和对应文章标题，以元数据形式呈现，不做大字封面。

每次交付 AI 生成封面时，报告所选 Bug 类型、自然秩序和钢蓝作用，为未来建立“第 1024 格”目录保留创作记录。
